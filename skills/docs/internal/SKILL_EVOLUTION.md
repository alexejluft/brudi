# Skill Evolution — Kontinuierliche Entwicklung

**Erstellt:** 2026-02-20
**Status:** Lebendiges System, nie "fertig"

---

## Die richtige Perspektive

Die 5.132 Zeilen heute sind **Version 0.1** — ein Startpunkt, kein Endpunkt.

Über Wochen und Monate wird dieses System:
- **Erweitert** — Neue Patterns aus echten Projekten
- **Verbessert** — Bessere Erklärungen, mehr Beispiele
- **Getestet** — Echte Anwendung zeigt Lücken
- **Optimiert** — Was funktioniert, was nicht
- **Verdichtet** — Überflüssiges entfernen, Essenz behalten

---

## Wann ich Skills erweitere

### Bei jedem Projekt
Nach jeder echten Arbeit (FairSplit, Zouk Website, neue Aufgaben):
1. Was habe ich gelernt?
2. Welcher Skill profitiert davon?
3. Fehlt ein Pattern? Ein Anti-Pattern?
4. War eine Erklärung unklar?

### Bei jedem Learning
Wenn ich etwas Neues lerne (Cron Sessions, Recherche):
1. Passt es in einen bestehenden Skill?
2. Braucht es einen neuen Skill?
3. Widerspricht es etwas Bestehendem?

### Bei Fehlern
Wenn etwas schief geht:
1. Hätte der Skill das verhindert?
2. Fehlt eine Warnung? Ein Anti-Pattern?
3. War die Anleitung missverständlich?

---

## Qualitätskriterien für Skills

Ein Skill ist gut wenn:
- [ ] Eine andere KI damit arbeiten kann
- [ ] Jedes Pattern aus echten Projekten stammt
- [ ] Anti-Patterns explizit genannt sind
- [ ] Beispiele echten Code zeigen
- [ ] Die Erklärungen das WARUM enthalten

Ein Skill muss verbessert werden wenn:
- [ ] Ich selbst etwas nachschlagen muss das nicht drin steht
- [ ] Ein Fehler passiert der vermeidbar gewesen wäre
- [ ] Die Erklärung zu oberflächlich ist
- [ ] Es neuere/bessere Ansätze gibt

## Das Ziel

In 6 Monaten soll dieses Skill-System:
1. **Umfassend** sein — Alles was für Award-level Webdev nötig ist
2. **Praxiserprobt** — Jedes Pattern in echten Projekten verwendet
3. **Installierbar** — Andere können es nutzen
4. **Einzigartig** — Etwas das es so nicht gibt

Das ist kein Sprint. Das ist ein Marathon.
